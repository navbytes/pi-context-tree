HOW — on each turn the extension sets the editor's borderColor to the current band's color (the same colors as the panel gauge). Input behaviour is unchanged: pi grafts all its handlers (escape, ctrl+d, model-switch, autocomplete, extension shortcuts) onto the supplied CustomEditor, which is the very class pi uses for its default editor.

TRADE-OFF (the honest bit) — pi reuses this same border for bash mode and thinking mode. So during those, the border briefly shows pi's bash/thinking color; it returns to the health color on the next turn. The health color is the resting state, not an exclusive owner of the border.

SCOPE — TUI only (no effect in RPC/headless, where setEditorComponent is a no-op). Complements — does not replace — the footer status ⎇ branch · ctx N% band and the panel gauge.
