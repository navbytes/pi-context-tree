HIGH
after a squash you regret — type /undo
› /undo ↩ undo: re-open 'fix-flaky-test' at its leaf? · the squash decision record stays in history (off-path, not deleted) · restores the branch's 20 turns to the active context [y]es [N]o
what it reverts, by last mutation type
squash / discard → re-open the branch (record + close marker kept, off-path) crop result/turn → pre-crop branch — the fat result / Q&A turn returns /branch → back to the parent; fork left dangling
Append-only is preserved — nothing is deleted. Each mutation already branches at an anchor; /undo is guided navigation back to that anchor (the recovery that's a tree-spelunking chore today, as one keystroke). Open question for you: last-mutation-only (my pick) vs a small undo stack.
